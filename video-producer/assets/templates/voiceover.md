# Voiceover

**连续讲故事** — 带观众沿一条线走，每个拐点挖透、收束、再接下一环。不是新闻通稿，不是事实串珠。事实句 `[Cxxx]`；分析 `[OPINION]`。

## Story rules (anti 新闻稿 / anti 戛然而止)

每个 major beat 写满四层再跳：**enter 承接 → evidence 证据 → mechanism 博弈/为什么 → landing 收束+留钩**

- 禁止硬切：此外 / 另外 / 与此同时 / 据悉 / 据报道 / 接下来再说
- 换 beat 用：所以到这里 / 带着刚才这一点 / 时间线再往后推一步 / 同一根链的下一环
- 全文至少 2 处 **callback**（回到刚才/前面那个限定词/同一份文件）
- 每个 beat 至少 **4 句口播** 再进入下一拐点

## Oral style

**要：** 短句、证据锚点（原文/公告/时间线/条款）、`[pause]` 在抛关键证据前

**不要：** 你想啊 / 其实吧 / 主观猜测口水；无 mechanism 和 landing 就跳下一段

## Draft

<!-- BEAT: hook / T001 -->

[enter]
很多人把整件事看成一次突然收紧——但如果把时间线往回拖一年，文件里早就有伏笔了。[C001]

[evidence + mechanism]
那次修订改的不是宣传口径，是谁还算“合规用户”。[C001b] 代理层从那一刻起就被动往窄里走。

[landing + carry]
所以问题不在后面的封号标题，而在定义被改窄了；带着这个限定词，我们看第二次收紧就不会觉得突兀。

<!-- BEAT: turn / T002 — enter 必须承接上一 beat carry -->

[enter]
时间线再往后推一步——同一份 ToS 里还有一处容易被跳过。[C002]

[evidence + mechanism]
条款写得很具体：foreign national，不是 everyone。[C002b] 很多误读，就从这里开始。

[landing + carry]
所以到这里，表面是“账号问题”，底下是“身份定义问题”——后面第三次震荡，只是同一逻辑的延伸。[OPINION]

<!-- BEAT: payoff — callback 前文 -->

[enter — callback]
回到开头那个限定词，把三次动作串起来，弧线就很清楚了。[C001][C002]

[landing]
整件事不是三个独立新闻，是一条定义越收越窄的链。[OPINION]
