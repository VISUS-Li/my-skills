## 从字幕里学什么（对照用）

读篇时标这五件事，然后换成**你自己的题**去搜、去写——不要换成「怎么说得像他」：

1. **开场**：他怎么用一件具体热闹/硬新闻把人拽进来？（谁、干了啥、为什么离谱）  
2. **还债顺序**：先丢结论再拆，还是按时间推？每一段结尾怎么自然接到下一段？  
3. **人物与专名出场**：生僻公司/人名出现时，他补了哪些履历、身份、钱、目标、时间点？有没有趣到能记住的细节？  
4. **当场含义**：数字/条款后，下一句怎么落到谁疼、谁赚、改什么判断？  
5. **深挖与收回**：哪一处用可看见的场面把机制讲透？何时半句拉回主线？**有没有情绪起伏**（惊讶、不信、逗）？

| 学（可迁移） | 拒（频道壳 / 浅模仿 / 教辅腔） |
|---|---|
| 聊天节奏、故事债、软转折、当场反应 | 「欢迎收听老×讲故事」及任何频道包装 |
| tip → 时间线/账单 → **按 tip 选** 1–2 处细讲 | 把每个出场公司各写一章；只堆语气词没证据 |
| 用履历与场面讲清专名 | 「你就记」「别听成黑话」「理解成××」拆词法 |
| 一个长场面换一条规则，然后回来 | 空洞金句串；强制分类菜单当目录 |
| 细讲写进主线，不贴「还有一层」标签 | 口播里出现「大白话」「活人味」等元讲解 |

## 研究时多找什么（写稿前的材料菜单）

不是每项必填；缺哪类就补哪类。只有 tip + 情绪时**先别写口播**。

| 材料 | 为什么需要 |
|---|---|
| 带日期的 tip 锚点 | 开场才能具体 |
| 有转折的时间线 | 「离谱」来自时序，不是态度 |
| 主角团履历（人/公司） | 专名才能故事化出场 |
| 规则底子（合同/许可/条款/标准） | 听众要懂卡在哪 |
| **邻居线 / 同批动作** | tip 常靠对照才看懂（同批备案、同业先例、政策变线） |
| 可对照的数字 + 含义 | 百分比 alone = 读稿机 |
| 值得细讲的缝（候选） | 写稿时按 tip 挑 1–2 个，不是名单全写 |
| 普通人冲击 | 落到谁付钱、谁被挡、工期怎么变 |
| 仍不确定边界 | 分开事实 vs 推断 |

锁定进 `research/event_genealogy.md` + source cards，口播按 `narrative-depth-copy.md` 写。

## 写稿协议（短）

1. `CATALOG.md` 按 tags 选 1–2 篇 → 标出上面五问（可极简）。  
2. **列 dig 计划（按 tip 判，不按人物 roster）**：本期 tip 是什么？哪 1–2 个节点最有知识量/戏剧性/误读价值？（邻居政策、机制缝、人物纠葛——谁服务 tip 写谁。）  
3. 按 `deep-research-and-script.md` 锁材料；人物履历不够就先补搜。  
4. 按 `narrative-depth-copy.md` 写**新**口播：嵌入主线细讲；留 TTS 情绪标点；正片不出现 skill 元讲解。  
5. 没听过老范的人也能顺着故事听懂 → 过；只会「像他说话」但时间线空 / 专名靠口令 / 像念新闻 → 重写。
