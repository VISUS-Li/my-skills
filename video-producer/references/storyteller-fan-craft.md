## 从字幕里学什么（对照用）

读篇时标这五件事，然后换成**你自己的题**去搜、去写——不要换成「怎么说得像他」：

1. **开场**：他怎么用一件具体热闹/硬新闻把人拽进来？（谁、干了啥、为什么离谱）  
2. **还债顺序**：先丢结论再拆，还是按时间推？每一段结尾怎么自然接到下一段？（不要求固定口头禅连接词）  
3. **人物与专名出场**：生僻公司/人名出现时，他补了哪些履历、身份、钱、目标、时间点？有没有趣到能记住的细节？  
4. **当场含义**：数字/条款后，下一句怎么落到谁疼、谁赚、改什么判断？  
5. **深挖与收回**：哪一处用可看见的场面把机制讲透？何时半句拉回主线？

| 学（可迁移） | 拒（频道壳 / 浅模仿 / 教辅腔） |
|---|---|
| 聊天节奏、故事债、软转折 | 「欢迎收听老×讲故事」及任何频道包装 |
| tip → 时间线/账单 → 选择性深挖 | 只堆「啊呢吧」却没有证据河 |
| 用履历与场面讲清专名 | 「你就记」「别听成黑话」「理解成××」拆词法 |
| 一个长场面换一条规则，然后回来 | 空洞金句串；强制分类菜单当目录 |

## 研究时多找什么（写稿前的材料菜单）

不是每项必填；缺哪类就补哪类。只有 tip + 情绪时**先别写口播**。

| 材料 | 为什么需要 |
|---|---|
| 带日期的 tip 锚点 | 开场才能具体 |
| 有转折的时间线 | 「离谱」来自时序，不是态度 |
| 主角团履历（人/公司） | 专名才能故事化出场：从哪来、融过资吗、目标是什么、现在卖什么 |
| 规则底子（合同/许可/条款/标准） | 听众要懂卡在哪 |
| 可对照的数字 + 含义 | 百分比 alone = 读稿机 |
| 一道值得深挖的缝 | 场面/类比只服务这一道缝 |
| 普通人冲击 | 落到谁付钱、谁被挡、工期怎么变 |
| 仍不确定边界 | 分开事实 vs 推断 |

锁定进 `research/event_genealogy.md` + source cards，口播按 `narrative-depth-copy.md` 写。

## 写稿协议（短）

1. `CATALOG.md` 按 tags 选 1–2 篇 → **在草稿或研究笔记里标出上面五问**（可极简，但要有）。  
2. 为自己的题列 dig 计划：时间线节点、必须出场的人物/公司背景、最多 1–2 道缝、可选场面、带走一句。  
3. 按 `deep-research-and-script.md` 锁材料（人物履历不够就先补搜，再写 VO）。  
4. 按 `narrative-depth-copy.md` 三道硬闸写**新**口播；写完用 Kill Checklist 自检，尤其查有无教辅拆词腔。  
5. 没听过老范的人也能顺着故事听懂 → 过；只会「像他说话」但时间线空 / 专名靠口令解释 → 重写。
