# Style DNA

Use one preset per first slice. Mix presets only after the first slice passes review.

## `vibemotion-dev-demo`

Use for developer tools, AI coding workflows, install/run/generate demos, repo walkthroughs, CLI proof, and product workflow videos.

- Colors: black, charcoal, off-white text, electric cyan/green accents, one warning accent such as red or amber.
- Background: dark grid, terminal grain, editor panels, file tree silhouettes, subtle scanline or grid pulse.
- Subtitle strategy: compact bottom subtitle, high contrast, never wider than the safe text area.
- Flower text: keyword pops for tool names, commands, generated files, time saved, and failure moments.
- SFX density: restrained but continuous; soft pop, keyboard tick, click, whoosh, light hit.
- Macro reset: every 8-10 seconds by switching room: grid intro -> editor -> terminal -> browser result -> generated artifact.
- Micro action: every 0.8-1.2 seconds; cursor, command typing, file reveal, redbox, panel slide, progress tick.
- Recommended recipes: `dark_grid_intro`, `terminal_proof`, `editor_code_zoom`, `screenshot_pushin_redbox`, `workflow_pipeline`, `audio_waveform_sync`, `conclusion_stamp`.
- Forbidden: plain command text on empty background, unanimated screenshots, decorative neon with no proof, full-screen subtitles as the main visual.

## `ai-chapingjun-system-explainer`

Use for AI product analysis, tool comparisons, industry phenomenon breakdowns, critique videos, and dashboard-driven explainers.

- Colors: blue-white system UI, dark analysis rooms, white cards, red critique marks, yellow warnings.
- Background: dashboard, product screenshot wall, metric board, layered cards, browser evidence stack.
- Subtitle strategy: short phrase captions; let charts/cards carry facts.
- Flower text: critique labels, turning words, numbers, suspicious claims, conclusion stamps.
- SFX density: medium-high for social rhythm; click, marker, card whoosh, glitch for warning, bass hit for conclusion.
- Macro reset: every 8-12 seconds by changing analysis room: product proof -> claim board -> compare table -> critique wall.
- Micro action: every 1.0-1.5 seconds; card stack, redbox, metric count, crop zoom, source stamp.
- Recommended recipes: `light_grid_concept_board`, `dashboard_room`, `critique_wall`, `data_card_compare`, `before_after_split`, `timeline_rewind`, `conclusion_stamp`.
- Forbidden: generic AI stock visuals, unsupported claims, charts without axes/labels, random glitch that does not mark a warning.

## `git-motion-edu-metaphor`

Use for technical teaching, Git concepts, code flows, time travel, branching, rollback, snapshots, and developer mental models.

- Colors: light grid or black grid, clean code colors, green additions, red deletions, purple/blue branch accents used sparingly.
- Background: grid board, phone/chat pane, Git graph lane, code editor, terminal, flow diagram.
- Subtitle strategy: concise bottom subtitle plus in-scene labels attached to graph nodes or code spans.
- Flower text: key concept words, commands, branch names, before/after states.
- SFX density: educational and precise; click, typing, node pop, whoosh, reverse whoosh, stamp.
- Macro reset: every 10-12 seconds by moving between metaphor rooms: chat -> Git graph -> code -> terminal -> timeline.
- Micro action: every 0.8-1.3 seconds; node growth, branch draw, diff highlight, command tick, rewind scrub.
- Recommended recipes: `git_graph_growth`, `timeline_rewind`, `terminal_proof`, `editor_code_zoom`, `phone_chat_sequence`, `svg_metaphor_scene`, `workflow_pipeline`.
- Forbidden: explaining abstract concepts only with text, branch diagrams that do not animate direction, code blocks too small to read.
