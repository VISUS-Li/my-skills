# 风格 DNA

首片只用一个预设。混预设须等首片过评审之后。

## `vibemotion-dev-demo`

用于开发者工具、AI 编程工作流、安装/运行/生成演示、仓库漫游、CLI 证明、产品工作流视频。

- 色彩：黑、炭灰、米白文字、电青/绿强调，一个警告强调色如红或琥珀。
- 背景：暗网格、终端颗粒、编辑器面板、文件树剪影、轻扫描线或网格脉冲。
- 字幕策略：底部紧凑字幕、高对比，永不超出安全文字区。
- 花字：工具名、命令、生成文件、省时、失败时刻的关键词弹出。
- SFX 密度：克制但连续；soft pop、键盘 tick、click、whoosh、轻 hit。
- 宏重置：每 8–10 秒换房：grid intro -> editor -> terminal -> browser result -> generated artifact。
- 微动作：每 0.8–1.2 秒；光标、命令打字、文件揭示、红框、面板滑入、进度 tick。
- 推荐配方：`dark_grid_intro`、`terminal_proof`、`editor_code_zoom`、`screenshot_pushin_redbox`、`workflow_pipeline`、`audio_waveform_sync`、`conclusion_stamp`。
- 禁止：空背景上的纯命令文字、无动画截图、无证明的装饰霓虹、全屏字幕当主视觉。

## `ai-chapingjun-system-explainer`

用于 AI 产品分析、工具对比、行业现象拆解、批评视频、仪表盘驱动讲解。

- 色彩：蓝白系统 UI、暗分析房、白卡片、红批评标记、黄警告。
- 背景：仪表盘、产品截图墙、指标板、分层卡片、浏览器证据叠。
- 字幕策略：短短语字幕；让图表/卡片承载事实。
- 花字：批评标签、转折词、数字、可疑主张、结论 stamp。
- SFX 密度：中高，社交节奏；click、marker、卡片 whoosh、警告用 glitch、结论用 bass hit。
- 宏重置：每 8–12 秒换分析房：product proof -> claim board -> compare table -> critique wall。
- 微动作：每 1.0–1.5 秒；卡片叠、红框、指标计数、裁切缩放、来源 stamp。
- 推荐配方：`light_grid_concept_board`、`dashboard_room`、`critique_wall`、`data_card_compare`、`before_after_split`、`timeline_rewind`、`conclusion_stamp`。
- 禁止：通用 AI 素材图、无支撑主张、无轴/标签的图表、不标警告的随机 glitch。

## `git-motion-edu-metaphor`

用于技术教学、Git 概念、代码流、时间旅行、分支、回滚、快照、开发者心智模型。

- 色彩：亮网格或黑网格、干净代码色、绿增、红删、紫/蓝分支强调慎用。
- 背景：网格板、手机/聊天窗、Git 图轨、代码编辑器、终端、流程图。
- 字幕策略：底部简洁字幕 + 挂在图节点或代码跨度上的场内标签。
- 花字：关键概念词、命令、分支名、前后状态。
- SFX 密度：教学且精确；click、打字、node pop、whoosh、reverse whoosh、stamp。
- 宏重置：每 10–12 秒在隐喻房之间移动：chat -> Git graph -> code -> terminal -> timeline。
- 微动作：每 0.8–1.3 秒；节点生长、分支描线、diff 高亮、命令 tick、回放 scrub。
- 推荐配方：`git_graph_growth`、`timeline_rewind`、`terminal_proof`、`editor_code_zoom`、`phone_chat_sequence`、`svg_metaphor_scene`、`workflow_pipeline`。
- 禁止：只用文字讲抽象概念、无方向动画的分支图、代码块太小读不清。
