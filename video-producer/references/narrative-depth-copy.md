# Narrative Depth Copy

**SSOT for Chinese analytical / news / product / industry VO.**  
Load with `references/deep-research-and-script.md` when drafting `outputs/script.md`. Other files link here — do not restate these rules.

## Voice default（必须靠齐）

分析/资讯类口播默认写成**老范讲故事式聊天讲解**：把一件事按**来龙去脉**讲清楚，让听众能复述、能听懂、能带走一句判断。

写稿前打开 `references/storyteller-fan-craft.md`，再从 `storyteller-fan-corpus/CATALOG.md` **只读 1–2 篇**字幕，学：

- 怎么用具体热闹/硬新闻开场，怎么先撂结论再还债  
- 人名、公司、专名怎么用**履历、身份、融资、目标、时间线**嵌进故事（不是课后名词表）  
- 数字/条款出现后怎么立刻落到谁疼、谁赚、改什么判断  
- 机制难处怎么用**可看见的场面**讲透，挖完怎么收回主线  
- 哪里有**当场反应**（离谱、拧巴、逗）和**情绪起伏**，读出来像真人在讲  

**学讲解深度、故事债与聊天节奏；不学频道壳 / CTA / 口头禅；禁止照搬原片措辞与事实。**

纯 UI 演示、无主张 motion test、或用户明确不要这种口播腔时，可跳过 corpus。

## Three Hard Gates（写稿 blocker）

### 1. 故事一条线（连贯优先）

听完能用大约一分钟复述：**热闹 tip → 谁干了啥/发生了什么 → 卡在哪 / 还不确定什么 → 带走什么。**

写稿要领：

- **一段接一段往下推**；段与段之间要有因果或转折（「但是」「那为什么」「讲完 A，还有 B」这类自然口气即可）。  
- 连接词是**可选润滑**，不是打卡项；不要为了凑「所以呢 / 但是呢」硬塞。  
- **分类框不能当脊柱**：先立「几类产品 / 几层架构」再填名单 → 失败。分类若必要，必须嵌在故事债里（因为发生了 X，才会分出货和房）。  
- 可以「先丢结论，再一层一层还债」；也可以按时间线往前推。二选一写透。  
- 短片常见形状：`tip 利害 → 人物/机构进场 → 时间线还债 → 卡点现场 → 落到听众 → 边界`。  
- **选择性深挖**：按本期 tip 和影响挑 **1–2 处**（见下节「深挖怎么选」），挖完半句收回主线。

只有态度口号、三点洞见、或抽象框架当目录 → fail。

> **内部术语**：workflow 里可写 `causal river` / `dig_worthy`。**口播正文里不要出现**「因果河」「换轨」「卡点」「还债」等制作术语，除非换成听众能直接听懂的现场说法。

### 2. 当场听懂（用背景讲，不解构）

重要数字 / 机构 / 人名 / 条款 / 陌生专名，**同一话轮内**讲清楚听众要知道的东西：

- **这是谁 / 这是什么**（履历、缘起、商业还是实验室、目标是什么）  
- **现在什么状态**（公开了吗、融了多少量级、能干什么）  
- **意味着什么**（谁便宜了、谁仍得干活、改了什么判断）

写入方式：**嵌进叙事**，像讲新闻人物出场。

| 好 | 差 |
|---|---|
| 「创办人里头有李飞飞……2024 年开了商业公司……融到大约……第一个产品叫 Marble」 | 「World Labs——你就记：它赌空间智能」 |
| 「官方写：草稿很快；高质量网格可能磨到一小时」→ 立刻接「工程会上还在问墙碰得动吗」 | 「别听成一句话交作业；你可以把它理解成样板间」 |
| 「设置里突然多了 Apple 智能入口，模型包也能下，几小时后又撤了——说明功能差不多做好了，监管手续还没走完」 | 「灯已经能亮，缺的是那枚章」（听众不知道什么灯） |

**比喻可以用，但同一口气里要把对象讲明白**：是工程侧、监管侧、还是商业侧？短口号式隐喻单独成句 → 听众会懵。

人尽皆知的词按受众可以不解释；第二次起不要再念说明书。

### 3. 聊天感 + 活人感（能读出声，像当面讲热闹）

必须能一口气读出声；像跟朋友讲刚发生的新闻，不是念通稿、不是念大纲、不是开小灶。

| 要有 | 避免 |
|---|---|
| 短句、自然问句、故事现场 | 书面长句、对仗金句海报体 |
| 有根据的当场反应：离谱 / 逗 / 拧巴 / 「别急着鼓掌」 | 冷静罗列事实、无材料的纯侃 |
| **TTS 友好的情绪标点**：`！？`、省略号 `……`、破折、短促独词句（「憋啊。」「那是猜，不是公告！」） | 全文句号平铺、无起伏 |
| 用工程会、评论区、设置页、乌龙推送等**具体场面**讲机制 | 为高级感硬造抽象隐喻骨架 |
| 软转折收回主线 | 「第一部分：……」式章节标题 |

**TTS 提示（可选写在稿末附录，不进正片口播）：** 在需要停顿、质疑、压节奏、钉边界的地方标情绪；问号表惊讶/不信，感叹表强调，省略号表转念或留白。

**听感自测：** 读给不懂行的朋友听——若觉得「你在念高级评论 / 念大纲 / 在教我怎么理解」→ fail。

## 深挖怎么选（按内容判断，不按名单）

深挖是为了让听众**有收获**，不是把出场人物各写一章。写稿前对照 `research/event_genealogy.md`，**按本期 tip 重新判断**该展开哪 1–2 处（研究阶段标记的 `dig_worthy` 只是候选，不是必写清单）。

优先从这些维度里挑**最有知识量、最能改判断、最有戏剧性**的：

| 维度 | 什么时候值得细讲 | 例子（Apple 智能备案题） |
|---|---|---|
| **tip 的邻居** | 同批公告、同规则、同行业动作能改读法 | 华为/小米等七家同批备案；988 款云侧 vs 端侧首次单列 |
| **规则/政策变线** | 听众不懂「为什么现在才发生」 | 生成式暂行办法；监管目光从云到手机 |
| **人物/公司纠葛** | 选型、谈判、反转、老本行 vs 新角色 | 百度一度接近又拧巴；蔡崇信确认阿里 |
| **机制缝** | tip 背后有常见误读 | 「进中国」≠ 已上线；备案 ≠ 开关 |
| **对照先例** | 一对事实比十句态度有用 | 三星国行早换百度；苹果补课入列 |

**嵌入主线讲**，不要为了「扩百度」「扩阿里」机械分章。人物故事、政策背景、同批名单，谁服务 tip 就写谁；其余一笔带过或留到 `still_unknown`。

短片默认 **1–2 处**深挖；若用户要更长稿，仍保持「每处挖完收回」，不要堆成人物百科。

## 口播正文里不要出现的写法

这些是**写稿说明 / skill 术语 / 元讲解**，容易显得机械、出戏：

- 「大白话就是……」「用人话说……」（直接说人话即可）  
- 「还有一层活人味 / 更有意思的是……」当结构标签  
- 「因果河」「换轨」「卡点」「story beat」等制作词  
- 未解释的短隐喻口号（「灯亮了章没盖」而不说明是设置页 vs 备案）  
- 附录里的 Kill Checklist、craft notes、TTS 表——**不要混进正片口播**

| 差 | 好 |
|---|---|
| 「大白话：你要备案才能开。」 | 「要对公众提供生成式 AI，得先走完备案。」 |
| 「还有一层活人味：百度并不弱。」 | 「很多人以为百度只会赶晚集——其实它 2013 年就在砸深度学习。」 |
| 「缺的是那枚章。」 | 「监管备案这道手续还没走完，所以入口亮了一下又关掉。」 |

教辅拆词腔仍一律重写：`你就记` / `别听成黑话` / `你可以把它理解成` / `简单来说就是` 等。

## 仍要避免的翻车（精简）

1. 先立抽象分类/隐喻骨架，再填事实。  
2. 专名、产品、融资连发，没有「因为 → 所以」。  
3. 金句串烧、空「不是 A 而是 B」当脊柱。  
4. 频道壳：欢迎收听、点赞小铃铛、Discord/付费、仿主持人口头禅。  
5. 事实与推断搅一锅（推断可以写，要和硬事实分开）。  
6. 为短而砍人物：不熟的公司/人若是主轴，用履历嵌进故事，别用口令糊弄。

类比**可选**：机制真的难懂时用，优先**具体场面**；不要为了「必须有一个主隐喻」硬塞。

---

## Ending

平常话钉住：发生了什么 + 意味着什么 + 还不确定什么。可钩回听众场景。  
禁止频道 CTA、空对仗落幕。

---

## Kill Checklist（任一失败则重写）

1. 复述不出来龙去脉（只有洞见/分类框）。  
2. 关键人名/机构/数字没有当场用背景或影响砸实。  
3. 读出声像念稿、念大纲、开小灶讲课；或含教辅拆词 / skill 元讲解（「大白话」「活人味」等）。  
4. 段落靠话题桶切换，故事接不动。  
5. 频道壳 / CTA，或无材料推断装成定论。  
6. 有比喻或短句口号，但同一话轮内没讲清对象（工程 / 监管 / 商业），听众会懵。  
7. 深挖像「人物百科」或「念新闻清单」，而不是服务 tip 的 1–2 处有收获展开。

## One-Line System Prompt

像老范跟朋友讲热新闻：先把热闹说具体，按来龙去脉往下讲；专名用履历和目标带进出场；数字和卡点落到谁疼、现场怎么拧巴；按 tip 挑 1–2 处最有料的地方细讲并收回；留一点情绪起伏给 TTS；别备课、别拆词、别把 skill 术语写进正片。
