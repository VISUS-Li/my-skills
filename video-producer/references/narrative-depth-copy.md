# Narrative Depth Copy

**SSOT for Chinese analytical / news / product / industry VO.**  
Load with `references/deep-research-and-script.md` when drafting `outputs/script.md`. Other files link here — do not restate these rules.

## Voice default（必须靠齐）

分析/资讯类口播默认写成**老范讲故事式聊天讲解**：把一件事按因果讲清楚，让听众能复述、能听懂、能带走一句判断。

写稿前打开 `references/storyteller-fan-craft.md`，再从 `storyteller-fan-corpus/CATALOG.md` **只读 1–2 篇**字幕，学：

- 怎么用具体热闹/硬新闻开场，怎么先撂结论再还债  
- 人名、公司、专名怎么用**履历、身份、融资、目标、时间线**嵌进故事（不是课后名词表）  
- 数字/条款出现后怎么立刻落到谁疼、谁赚、改什么判断  
- 机制难处怎么用**现场画面**讲透，挖完怎么收回主线  

**学讲解深度、故事债与聊天节奏；不学频道壳 / CTA / 口头禅；禁止照搬原片措辞与事实。**

纯 UI 演示、无主张 motion test、或用户明确不要这种口播腔时，可跳过 corpus。

## Three Hard Gates（写稿 blocker）

### 1. Causal River（因果河 · 连贯优先）

听完能用大约一分钟复述：**热闹 tip → 谁干了啥/发生了什么 → 卡在哪 / 还不确定什么 → 带走什么。**

硬规矩：

- **故事一条线往下推**；段与段之间要有因果或转折关系（可以是「但是」「那为什么」「讲完 A，还有 B」这类自然口气）。  
- **不要**为了满足连接词配额而硬塞「所以呢 / 但是呢 / 那问题来了」——连接词是可选润滑，不是打卡项。  
- **禁止话题桶当脊柱**：先立抽象分类框（几类产品、几层房间、买卖刀切什么菜），再往里塞名单。分类若必要，必须嵌在故事债里（因为发生了 X，才会分出货和房），不能当目录页。  
- 可以「先丢结论，再一层一层还债」；也可以按时间线往前推。二选一写透即可。  
- 短片常见形状：`tip 利害 → 人物/机构进场 → 时间线还债 → 卡点现场 → 落到听众 → 边界`。深挖最多 **1–2 处**，挖完半句收回。

只有态度口号、三点洞见、或隐喻框架当目录 → fail。

### 2. Instant Comprehension（当场听懂 · 用背景讲，不解构）

重要数字 / 机构 / 人名 / 条款 / 陌生专名，**同一话轮内**讲清楚听众要知道的东西：

- **这是谁 / 这是什么公司或产品**（履历、创办缘起、商业还是实验室、目标是什么）  
- **现在什么状态**（公开了吗、融了多少量级、能干什么）  
- **意味着什么**（谁便宜了、谁仍得干活、改了什么判断）

写入方式：**嵌进叙事**，像讲新闻人物出场；**禁止**课后定义体与拆词教学法。

| 要 | 不要 |
|---|---|
| 「创办人里头有李飞飞……2024 年开了商业公司……融到大约……第一个产品叫 Marble」 | 「World Labs——你就记：它赌空间智能」 |
| 「World API 把同一套能力接进别的软件和游戏」 | 「别听成黑话，就是把能力做成接口」 |
| 「官方写：草稿很快；高质量网格可能磨到一小时」立刻接「工程会上还在问墙碰得动吗」 | 「别听成一句话交作业；你可以把它理解成样板间」 |

人尽皆知的词按受众可以不解释；第二次起不要再念说明书。

坏：抛一串产品名/涨幅就下一段。  
坏：每个名词都加「你就记 / 别听成 / 简单说就是」的教辅括号。  
好：名词一露面，用一段背景或现场把份量砸实。

### 3. Oral Chat（聊天故事，不念稿不备课）

必须能一口气读出声，像当面跟人讲刚发生的热闹。

| 要有 | 不要有 |
|---|---|
| 短句、自然问句、故事现场 | 书面整饬长句、对仗金句海报体 |
| 偶尔自然称呼听众可以，**不强制**堆「你看 / 那你说 / 大家注意」 | 「综上所述」「基于上述」「值得注意的是」当脊柱 |
| 有根据的当场反应（离谱 / 逗 / 拧巴） | 冷静念新闻通稿、或无材料纯侃 |
| 软转折收回主线 | 章节标题转场：「第一部分：……」 |
| 用工程会、评论区、工地学徒等**可看见的场面**讲机制 | 为高级感硬造抽象隐喻骨架 |

**听感自测：** 读给不懂行的朋友听——若觉得「你在念高级评论 / 念大纲 / 开小灶讲课」→ fail。

## 明确禁止（常见翻车）

1. **为高级而高级**：先立抽象隐喻/分类骨架，再填事实。  
2. **清单代讲解**：专名、产品、融资连发，没有「因为 → 所以」。  
3. **金句灌装**：对仗、空「不是 A 而是 B」串烧当脊柱。  
4. **频道壳**：欢迎收听××、点赞小铃铛、Discord/付费、仿主持人口头禅。  
5. **事实与推断搅一锅**：推断可以有，但须和硬事实拆开。  
6. **教辅拆词腔**（本次迭代重点）：`你就记` / `别听成黑话` / `别听成……` / `你可以把它理解成` / `你就当作是` / `简单来说就是` 当专名落地主力 → 一律重写。专名靠故事背景落地，不靠元讲解。  
7. **为短而砍人物**：听众不熟的机构/人若是故事主轴，宁可多两句履历，不可用口令式简称糊弄。

类比**可选**：只在机制真的难懂时用，且优先**具体场面**（工程会三问、工地学徒），不要为了「必须有一个主隐喻」硬塞装修/样板间等教辅喻。

---

## Ending

平常话钉住：发生了什么 + 意味着什么 + 还不确定什么。可钩回听众场景。  
禁止频道 CTA、空对仗落幕。

---

## Kill Checklist（任一失败则重写）

1. 复述不出因果河（只有洞见/分类框）。  
2. 关键人名/机构/数字没有当场用背景或影响砸实。  
3. 读出声像念稿、念大纲、开小灶讲课，或充满教辅拆词。  
4. 段落主要靠话题桶切换，故事接不动。  
5. 频道壳 / CTA，或无材料推断装成定论。

---

## One-Line System Prompt

像老范跟朋友讲热新闻：先把热闹说具体，按因果还债；人名公司用履历和目标带进出场；数字和卡点落到谁疼、现场怎么拧巴；别备课、别拆词、别为高级感立分类架。
