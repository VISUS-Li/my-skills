# Explainer Visual Metaphor Library

Use this library to avoid generic cards and bullet lists. Pick one dominant metaphor per scene and reuse it so the video feels designed.

## AI / text generation metaphors

- **Factory / machine**: input card enters machine, gears/modules process, output appears.
- **Conveyor belt**: examples move left-to-right, rejected items drop, successful items get stamped.
- **Scanner / camera**: Chinese glyph is scanned into boxes, edges, strokes, features.
- **CLIP bridge**: text island and image island connected by semantic vector bridge.
- **Black box**: proprietary model; inputs/outputs visible, internals locked.
- **Glyph module**: separate component that injects exact character shape into image generation.
- **OCR judge**: tiny camera or robot compares rendered text with requested text.
- **AnyText workshop**: text layout board + glyph guide + diffusion machine.
- **Model leaderboard**: product cards in a game HUD with ranks and status pills.

## Devices and environments

- Desk with phone, stickers, plush/mascot: creator-native and tactile.
- Browser dashboard: serious tool/product proof.
- Whiteboard lab: teaching mechanisms.
- Model factory: pipelines, modules, gauges, switches.
- Museum/archive cards: historical examples or old model failures.
- Game UI/HUD: rankings, success/fail states, level-up explanations.

## Motion ideas by metaphor

- Factory: slot insert, belt move, mechanical hum, gauge fill, output hatch pop.
- Scanner: scanning beam, bbox draw, tick marks, camera shutter, OCR beep.
- Bridge: arrow pulse, node glow, vector bars sliding, lock click.
- Black box: lock shut, question mark orbit, red internal-unavailable stamp.
- Glyph module: strokes draw in order, snap-to-grid, character outline locks, success chime.
- Leaderboard: cards shuffle, rank badges pop, small trophy sparkle, number count-up.

## Reusable composition patterns

- `Problem wall`: gallery of failures fills the background, red stamp lands on top.
- `Mechanism cutaway`: camera zooms inside a machine to reveal modules.
- `Before / after lab bench`: left messy outputs, right corrected outputs.
- `Semantic map`: text and images as nodes arranged on a coordinate plane.
- `Device demo`: content lives inside a phone/laptop, annotations orbit outside.
- `Evidence carousel`: 3-5 examples move like cards on a rail.
- `Mini world`: tiny mascot or robot walks between modules to guide attention.
