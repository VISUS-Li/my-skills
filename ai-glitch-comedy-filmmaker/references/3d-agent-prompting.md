## 1. Agent 能力假设

默认 Cursor 等编程 Agent 会出现：

- 只写代码，不完成场景挂载；
- 擅自重构现有控制器；
- 忽略 Inspector/节点参数；
- 把故意设计的错误“修正常”；
- 添加不必要依赖、包和资产；
- 一次改动过多，导致无法判断问题来源；
- 使用抽象描述代替可观察结果；
- 声称完成，但没有给出测试步骤。

因此不要交付一个巨大提示词。将功能拆成 2–5 个串行任务，每个任务结束时游戏都能运行并验证。

## 2. 任务拆分原则

一个任务只完成一个可见结果：

- 任务 A：建立正常第三人称移动对照。
- 任务 B：实现故意的身体/肢体位移脱钩。
- 任务 C：让碰撞体留在原位，形成可利用的穿墙侦察。
- 任务 D：加入一键切换正常/失控和复位。
- 任务 E：加入录制用 UI 与自由摄像机。

不要在同一任务里同时制作新角色、动画系统、敌人 AI、任务逻辑和剪辑 UI。

## 3. 全局约束块

每个项目先提供全局约束：

- 使用现有引擎、版本和语言；未知时明确假设。
- 只修改列出的文件和场景；确需新增文件时先列出。
- 不安装第三方包，不下载资产，不替换渲染管线。
- 使用现有角色；没有时使用胶囊、方块和基础骨骼占位。
- 不重构无关系统。
- 不追求完整游戏、通用框架或生产级架构。
- 每次改动保持项目可运行。
- 故意失控行为属于需求，不要自动修复。
- 所有关键参数暴露在 Inspector/配置中。
- 提供正常模式、失控模式、复位和调试显示。
- 完成后必须列出修改文件、挂载步骤、测试步骤和已知限制。

## 5. 视觉目标写法

不要写：

- “角色跑得很诡异。”
- “做一个搞笑动画。”
- “动作要像幽灵。”

改写为可观察的 3D 条件：

- 根节点以 3.5 m/s 沿地面移动。
- 胸部模型离地 0.4 m，不播放标准跑步动画。
- 左右手脚分别相对骨盆延迟 0.15–0.35 秒。
- 四肢与身体最大距离限制为 1.2 m，避免完全飞出镜头。
- 四肢落地时使用轻微弹性，不影响根节点速度。
- 第三人称摄像机保持 6 m 距离和 2.5 m 高度，能同时看到身体与四肢。

描述时至少包含：对象、相对位置、速度/延迟、尺度、碰撞关系、镜头可读性。

## 6. 3D 场景与组件写法

场景层级必须用明确路径或等价节点表示：

```text
PrototypeScene
├── Environment
│   ├── Ground
│   ├── Wall_Normal
│   └── Wall_Test
├── PlayerRoot
│   ├── CollisionBody
│   ├── VisualRoot
│   │   ├── Torso
│   │   ├── Head
│   │   ├── Arm_L
│   │   ├── Arm_R
│   │   ├── Leg_L
│   │   └── Leg_R
│   └── FollowCameraTarget
├── DragonNPC
├── GoalTrigger
└── DebugCanvas
```

每个组件写明：

- 挂在哪个对象；
- 负责什么；
- 读取什么输入；
- 修改什么状态；
- 哪些参数可调；
- 正常模式和失控模式差异。

不确定引擎 API 时，先给引擎无关逻辑，再给目标引擎实现建议。不要虚构不存在的现有文件或节点。

## 7. 验收与反例

验收必须可由用户手动测试：

- 按 W/摇杆 3 秒，身体前进约 10 米。
- 四肢至少与身体分离 0.6 米，但不超过 1.2 米。
- 摄像机同一画面能看到身体、至少三只肢体和前方墙体。
- 切换 `chaos_enabled=false` 后恢复普通整体移动。
- 按 R 后角色、四肢和摄像机回到初始状态。
- 连续测试 5 次均能出现相同效果。

失败反例要具体：

- 四肢只轻微抖动，远景看不出分离。
- 根节点和碰撞体一起穿墙，无法形成“模型穿墙但实体留下”的后续玩法。
- 四肢无限飞走，三秒后离开镜头。
- Agent 用新的完整动画包替代现有粗糙模型。
- Agent 把分离视为 BUG 并自动恢复骨骼绑定。

## 8. 完整示例：自然跑步失败

### 任务目标

为现有第三人称角色加入“自然跑步”功能的故意失败版本。视觉目标不是随机 ragdoll，而是稳定、可录制的身体与四肢脱节：躯干像幽灵一样平滑悬浮，四肢在后方追赶。

### 精确约束

- 不替换现有玩家输入和摄像机。
- 不下载动画或模型。
- `PlayerRoot` 继续负责真实位置和碰撞。
- `VisualRoot` 跟随 `PlayerRoot`，但四肢子对象使用独立延迟位置。
- 躯干高度固定为地面上方 0.4 m。
- 四肢延迟参数默认 0.25 秒，最大偏移 1.0 m。
- 正常模式下所有肢体恢复局部初始位置。
- 失控模式下不影响玩家转向、速度和胜负触发。
- 提供 `chaos_enabled`、`limb_lag_seconds`、`max_limb_distance`、`torso_hover_height`。

### 画面参考

错误第一次出现时，镜头必须先看到角色整体，然后角色前进，胸部向前滑，手脚像被绳子拖着。不要让肢体直接消失，也不要让角色倒地不能操作。

### Agent 完成回复格式

1. 修改/新增文件。
2. 场景对象和挂载位置。
3. Inspector/节点参数。
4. 操作测试步骤。
5. 正常模式与失控模式切换方法。
6. 已知限制。
7. 未完成事项；没有则写“无”。


## 成片创作与技术实现必须分开

当 Agent 同时被要求生成游戏实现和视频方案时：

- 先输出可直接录音的完整旁白总稿。
- 再输出 `creative_storyboard`，其中不得出现内部状态名、参数或调试字段。
- 最后输出 `technical_capture_plan` 和编码任务。
- 禁止用 `VO1`、`可选旁白` 或“无旁白”大量填充。
- 任何 15 秒以上镜头都必须有实际说出的旁白/对白，或拆成视觉揭示和后果两个镜头。
- 技术实现完成不代表创作交付完成；旁白密度和分镜可读性必须单独验收。
