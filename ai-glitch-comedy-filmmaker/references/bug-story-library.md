# 粗糙 3D 游戏失控素材库

这不是故事清单，而是“正常需求 → Agent 常见错误 → 可见画面 → 可玩后果 → 吐槽方向”的组合库。

## 1. 角色与动画

| 正常需求 | 3D 错误结果 | 可玩后果 | 吐槽方向 |
|---|---|---|---|
| 自然跑步 | 躯干悬浮，四肢延迟拖行 | 模型与碰撞体脱钩，可穿墙侦察 | “自然”被重新定义为不受人体约束 |
| 跳得更高 | 角色模型变高而不是跳跃高度增加 | 头部越过围墙，但脚仍在原地 | 高，主要指角色高度 |
| 改善转身 | 身体保持朝前，头部独立旋转 360 度 | 可以看后方但瞄准仍朝前 | 视野开阔，主要是颈部层面 |
| 加入受击反应 | 每次受击都触发完整 ragdoll，但输入继续移动 | 玩家像软体袋一样冲过敌人 | 反应非常充分，恢复步骤暂未安排 |
| 动画更流畅 | 所有动画状态同时混合 | 角色边跑边坐边攻击 | 每个部门都参与了这次动作 |

## 2. 物理与碰撞

| 正常需求 | 3D 错误结果 | 可玩后果 | 吐槽方向 |
|---|---|---|---|
| 减少穿墙 | 墙体开始跟随玩家 | 玩家永远处于室内 | 墙没有被穿过，它主动保持距离 |
| 物体更有重量 | 小箱子质量变成 1000，玩家被箱子拖走 | 箱子成为移动武器 | 货物的存在感得到加强 |
| 增加弹性 | 所有碰撞体像弹力球 | 玩家利用反弹跳关 | 地面开始拒绝长期接触 |
| 稳定堆叠 | 物体被约束到同一坐标 | 一堆箱子融合成巨大团块 | 物流完成了高度统一 |
| 防止角色跌倒 | 角色永远保持竖直，但脚离地 | 可在斜坡和空中站立 | 稳定，主要是不承认重力 |

## 3. NPC 与智能

| 正常需求 | 3D 错误结果 | 可玩后果 | 吐槽方向 |
|---|---|---|---|
| 智能追踪玩家 | NPC 追踪玩家上一帧位置 | 形成蛇形队列 | 他们很尊重历史 |
| 攻击最近敌人 | NPC 攻击最近对象，不检查阵营 | 敌人互殴、任务目标被攻击 | 没有偏见，所有人机会平等 |
| 守卫更警觉 | 所有守卫共享一个视线结果 | 一人看到，全图追杀 | 情报系统非常高效 |
| 自动避障 | NPC 把玩家视为障碍 | NPC 礼貌绕开玩家，永不攻击 | 敌意被路径规划解决了 |
| 跟随队长 | 每个 NPC 都认为自己是队长 | 队伍向不同方向出发 | 组织完成了扁平化管理 |

## 4. 任务与胜负

| 正常需求 | 3D 错误结果 | 可玩后果 | 吐槽方向 |
|---|---|---|---|
| 救出公主 | 救援区绑定最近对象，恶龙最近 | 玩家救走恶龙 | 目标很明确，只是身份不重要 |
| 消灭所有敌人 | 系统删除所有带角色标签的对象 | 玩家也消失，任务完成 | 敌我问题得到根本解决 |
| 到达最高点 | 系统提高角色 Y 坐标 | 玩家升入天空无法停止 | 登顶过程被大幅简化 |
| 收集十个金币 | 同一个金币触发器重复计数 | 玩家站着不动完成收集 | 重复利用也是一种环保 |
| 保护基地 | 基地被设为无敌并不可交互 | 玩家也进不去 | 安全措施成功阻止了所有人 |

## 5. 摄像机、UI 与声音

| 正常需求 | 3D 错误结果 | 可玩后果 | 吐槽方向 |
|---|---|---|---|
| 镜头更有动感 | 摄像机跟随最近刚体 | 被飞出的手臂带走 | 镜头选择了更有行动力的主角 |
| 任务箭头更智能 | 箭头指向最近对象 | 箭头持续指向玩家自己 | 系统鼓励从认识自己开始 |
| 音效匹配动作 | 音效绑定速度 | 漂浮角色发出密集跑步声 | 身体没跑，音效替他跑了 |
| 提升可读性 | 所有碰撞框永久显示 | 调试线成为画面主体 | 游戏选择公开全部内部结构 |
| 增加胜利反馈 | 每个小事件都播放胜利音 | 玩家走一步庆祝一次 | 系统对过程给予充分肯定 |

## 6. 组合公式

优先组合一主一辅：

- 动画错位 + 碰撞错位：形成模型穿墙、实体留后。
- NPC 感知 + 阵营标签：形成敌人误认同类。
- 最近目标 + 救援触发：形成救错对象。
- 极端物理 + 任务区域：形成弹飞过关但错过触发。
- 摄像机跟错 + 肢体分离：形成镜头追随一条腿。

每个组合至少写清：

1. 玩家怎样触发；
2. 画面怎样一眼看懂；
3. 玩家怎样利用；
4. 收益与代价；
5. 怎样稳定复现；
6. 哪一句 AI 承诺可以被冷面重新定义。
